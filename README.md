# 🚀 HCM Open API 数据集成与自动化对账引擎 (Final Edition)

本脚手架项目聚焦于大型企业 HCM（人力资本管理）系统实施中的核心痛点——**异构系统数据流转与薪酬合规风控**。通过 Python 面向对象设计，实现跨系统考勤数据的安全拉取、自动化清洗拦截，以及向薪酬核算模块的无缝下发。

---

## 🎯 业务背景与实施价值 (Business Context)

在实际实施交付中，**薪酬核算模块**是风险最高、最容易引发客户投诉的环节。算薪公式的准确性，高度依赖于前置模块（时间管理、绩效管理）的数据质量。
面对客户可能存在的外部独立打卡系统，依靠人工 Excel 导入极易出现漏算错算。本项目以代码赋能实施，核心实现以下业务价值：

1. **变被动为主动的风控前置**：改变传统实施中“先导入、再报错、最后查错”的被动局面。在数据流向薪酬引擎前，建立业务规则网关，自动识别并阻断严重旷工、超长加班等异常数据。
2. **闭环对账机制**：自动生成面向 HR 业务端的可视化 `.xlsx` 审计对账单，明确区分“系统自动放行数据”与“人工冻结复核数据”，消除数据黑盒。
3. **提升交付专业度**：通过标准化的 API 集成方案，取代手工搬运，大幅提升系统上线后的稳定运行质量。

---

## 🏗️ 核心架构与企业级特性 (Enterprise Features)

本项目不仅打通了业务流，在代码工程化标准上也进行了严格的设计：

* **🛡️ 数据脱敏合规 (Data Masking)**：日志系统引入了动态脱敏算法。在拦截异常记录（如旷工预警）打印日志时，自动隐去员工真实姓名（如 `张三` -> `张*`），严格遵守实施交付的数据安全合规红线。
* **🔁 高可用容错重试 (Resilience)**：针对客户现场可能出现的 API 网关网络抖动，内置了基于 `MAX_RETRIES` 的自动重试机制。在首次抓取超时失败时，系统会自动进行退避重试，保障核心数据同步任务的高可用性。
* **🔐 生产配置解耦 (Decoupling)**：采用 `.env` 环境变量文件管理 API 接口地址、Token 和敏感凭证，实现代码逻辑与生产配置的安全隔离。
* **📝 全链路审计留痕 (Audit Logging)**：通过 Python 标准 `logging` 模块，将每一步网络请求、校验规则拦截、同步结果记录至本地 `.log` 文件，确保后期运维有源可溯。
